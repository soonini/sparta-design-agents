# SPARTA 브랜드 디자인 에이전트 시스템

> 작성일: 2026-02-20 · 작성자: 문수진

---

## 한 줄 요약

검수 기준을 지킨 퍼널 콘텐츠를 만들기 위한 실무 가이드가 없어서, PDF를 구조화하고 제작/검수를 자동화하는 AI 에이전트 시스템을 만들었습니다.

---

## 1. 배경

### 목표

검수 가이드 기준을 충족하는 퍼널 콘텐츠 제작

### 현실

- 브랜드 가이드라인 PDF(132페이지)는 **완성된 결과물** 중심의 컨셉 문서
- "이렇게 만들어라"는 있지만, "어떻게 만들어라"는 없음
- 콘텐츠 타입별 텍스트 구성, 포맷별 배치 규칙, 컬러 조합 방법 같은 **실무 가이드 부재**

### 기존 시도

가이드라인 최신화 → PDF의 결과물을 해석 · 취합해서 실무 가이드를 만드는 작업

### 전환 계기

> 이미 완성된 결과물을 역으로 해석해서 실무 가이드를 만드는 것 자체가 무의미하게 느껴졌습니다.
> 어차피 PDF를 구조화하는 작업이 필요하다면, 사람이 매번 참조하는 문서가 아니라 AI가 자동으로 적용하는 시스템으로 만들자.

---

## 2. 접근 방식

### 기존

```
PDF(결과물) → 사람이 해석 → 사람이 적용 → 사람이 검수
```

### 변경

```
PDF(결과물) → 구조화된 스펙 → AI 자동 적용 → AI 자동 검수
                                    ↑
                             사람은 확인/승인만
```

### 핵심 차이

| 구분 | 기존 (PDF 최신화) | 변경 (에이전트 시스템) |
|------|-------------------|----------------------|
| 규격 참조 | 132페이지에서 직접 검색 | AI가 자동 참조 |
| 검수 기준 | 사람마다 다른 해석 | 32개 항목 정량 판정 |
| 실수 발견 | 사후 검수 | 제작 단계에서 실시간 준수 |
| 피드백 | 구두 전달, 휘발 | 자동 축적 · 패턴 분석 |
| 가이드 개선 | 수동 업데이트 | 데이터 기반 개선 제안 |

## 3. 산출물

### 3-1. 브랜드 스펙 구조화

PDF 132페이지를 AI가 이해할 수 있는 3개 문서로 변환

| 문서 | 역할 | 내용 |
|------|------|------|
| 컴포넌트 라이브러리 | 브랜드의 사전 | 컬러 코드, 로고 규격, 폰트 사양, 그래픽 규칙 등 모든 수치 |
| 통합 가이드라인 | 브랜드의 문법 | 콘텐츠 타입별 · 포맷별 · 브랜드별 적용 규칙, DO/DON'T |
| 검수 체크리스트 | 브랜드의 시험지 | 32개 검수 항목 + 심각도 3단계 + PASS/FAIL 판정 기준 |

> PDF에서 "충분한 여백을 확보하세요" → "T자 기준 클리어 스페이스, 최소 크기 Primary 22px / Stacked 45px"
> 이런 식으로 컨셉을 실무 스펙으로 전환한 것.

### 3-2. AI 에이전트

| 에이전트 | 역할 |
|---------|------|
| 제작 에이전트 | 요청 분석 → 텍스트 시안 → 승인 후 HTML/CSS 생성 → 셀프 검수 |
| 검수 에이전트 | 이미지/HTML 분석 → 32개 항목 순차 검수 → PASS/FAIL 판정 + 수정 방안 |

### 3-3. 슬래시 커맨드 (4종)

| 커맨드 | 용도 | 사용 예시 |
|--------|------|----------|
| `/sparta-design` | 디자인 제작 | `/sparta-design 내배캠 신규 과정 런칭 배너` |
| `/sparta-review` | 디자인 검수 | `/sparta-review ./banner-v2.png` |
| `/sparta-feedback` | 피드백 기록 | `/sparta-feedback 로고 여백이 좁아 보입니다` |
| `/sparta-learn` | 피드백 분석 | `/sparta-learn` |

### 3-4. 피드백 루프

```
제작/검수 → 피드백 자동 축적 → 패턴 분석 → 가이드라인 개선 제안 → 검토/승인 → 반영
```

- 위반 빈도 TOP 5, 모호한 기준, FAIL 비율 추이 등 데이터 기반 분석
- 사내 평가 체크리스트(퀄리티/브랜드/그로스 3영역 8항목) 통합 완료

## 4. 기대 효과

### 단기

- **제작 속도** — 가이드라인 찾아보는 시간 제거, 규격 자동 적용
- **검수 일관성** — 주관적 판단 → 32개 항목 정량 판정
- **온보딩** — 새 팀원도 커맨드 하나로 가이드라인 준수 콘텐츠 제작 가능

### 장기

- **가이드라인 진화** — "자주 위반되는 규칙", "모호한 기준" 을 객관적으로 파악
- **팀 학습 효과 측정** — FAIL 비율 추이로 브랜드 이해도 변화 확인

## 5. 오해 방지

| 오해 | 실제 |
|------|------|
| 디자이너 대체 | 규격 확인 자동화 도구. 크리에이티브 판단은 사람의 영역 |
| 피그마 대체 | HTML/CSS 시안은 초안 · 레퍼런스 용도. 최종 산출물은 기존 워크플로우 |
| PDF 불필요 | PDF는 원본(source of truth) 유지. 시스템은 PDF를 실무용으로 변환한 것 |
| 완성된 시스템 | v1. 팀 피드백으로 함께 개선해야 하는 단계 |

## 6. 요청 사항

### 우선

1. **체험** — Claude Code 설치 후 `/sparta-design`, `/sparta-review` 직접 사용
2. **피드백** — 부정확한 규칙, 빠진 항목, 불편한 점 공유

### 이후

3. **검수 기준 합의** — 32개 체크리스트 항목 적절성 팀 리뷰
4. **적용 범위 결정** — 어떤 콘텐츠부터 시스템 활용할지 논의
5. **피드백 루프 운영** — 2주 단위 분석 결과 공유 및 개선안 검토
