# 슬랙 메시지 (복사용)

---

## 채널 공유용

```
안녕하세요, 브랜드 디자인 관련 공유드립니다.

[배경]
검수 기준을 지킨 퍼널 콘텐츠를 만들려면 실무 가이드가 필요한데, 현재 브랜드 PDF는 완성 결과물 중심의 컨셉 문서라 실무 적용 가이드가 없는 상태입니다.
가이드라인 최신화를 진행하려 했지만, 결국 결과물을 역으로 해석해서 취합하는 작업이라 방향을 바꿨습니다.

[한 것]
PDF를 AI가 이해할 수 있는 구조화된 스펙으로 변환하고, 이를 기반으로 제작/검수를 자동화하는 에이전트 시스템을 만들었습니다.

• /sparta-design — 가이드라인 준수 디자인 시안 + HTML/CSS 자동 생성
• /sparta-review — 기존 디자인 32개 항목 자동 검수 (PASS/FAIL 판정)
• /sparta-feedback, /sparta-learn — 피드백 축적 → 분석 → 가이드라인 개선

디자이너 대체가 아니라, 로고 크기·여백·컬러 조합 같은 반복적 규격 확인을 자동화하는 도구입니다.

자세한 내용은 아래 문서에 정리했습니다.
→ [PROPOSAL.md 링크]

시간 되실 때 읽어봐 주시고, 의견 편하게 공유해 주세요 🙏
```

## 리더 DM용

```
안녕하세요, 브랜드 가이드라인 관련 작업 공유드립니다.

[배경]
목표: 검수 기준 충족하는 퍼널 콘텐츠 제작
현실: 브랜드 PDF는 완성 결과물 중심 컨셉 문서 → 실무 가이드 부재
기존 시도: 가이드라인 최신화 → PDF 결과물을 해석·취합해서 실무 가이드를 만드는 작업
전환 계기: 결과물을 역으로 해석하는 것 자체가 무의미 → AI가 한 번 구조화하고 이후 자동 적용하는 시스템으로 전환

[산출물]
1. 브랜드 스펙 구조화 — PDF 132p를 구조화된 실무 스펙 3종으로 변환 (컴포넌트 라이브러리 / 통합 가이드라인 / 검수 체크리스트 32항목)
2. AI 에이전트 — 제작 (시안→코드→셀프검수) + 검수 (32항목 PASS/FAIL 판정)
3. 슬래시 커맨드 4종 — /sparta-design, /sparta-review, /sparta-feedback, /sparta-learn
4. 피드백 루프 — 피드백 자동 축적 → 패턴 분석 → 가이드라인 개선 제안
5. 사내 평가 체크리스트 통합 — 퀄리티/브랜드/그로스 3영역 8항목

[기대 효과]
• 제작 속도 — 규격 자동 적용, 가이드라인 검색 시간 제거
• 검수 일관성 — 주관적 판단 → 정량 판정
• 온보딩 — 커맨드 하나로 가이드라인 준수 콘텐츠 제작
• 가이드라인 진화 — 데이터 기반 개선

[요청]
• 자세한 문서: [PROPOSAL.md 링크]
• 저장소: [GitHub 링크]
• 팀 미팅에서 같이 논의할 수 있으면 좋겠습니다.

궁금한 점이나 우려 사항 편하게 말씀해 주세요!
```
