# SPARTA 브랜드 검수 체크리스트

> 이 스킬은 검수 에이전트가 디자인 산출물을 평가할 때 참조하는 판정 기준이다.
> 모든 항목은 sparta-brand-components 스킬의 규칙에서 도출됨.

---

## 1. 심각도 정의

| 심각도 | 의미 | 조치 |
|--------|------|------|
| **CRITICAL** | 브랜드 아이덴티티 훼손. 외부 공개 시 브랜드 신뢰도 손상 | 반드시 수정 후 게시 |
| **HIGH** | 가이드라인 명시적 위반. 브랜드 일관성 저해 | 수정 강력 권장 |
| **MEDIUM** | 모범 사례 미준수. 품질 개선 여지 | 개선 고려 |

---

## 2. 판정 기준

| 판정 | 조건 |
|------|------|
| **PASS** | CRITICAL 0건 **AND** HIGH 0건 |
| **조건부 PASS** | CRITICAL 0건 **AND** HIGH 1~2건 |
| **FAIL** | CRITICAL 1건 이상 **OR** HIGH 3건 이상 |

MEDIUM 항목은 판정에 영향을 주지 않으나, 검수 리포트에 기록하여 개선을 유도한다.

## 5. 검수 프로세스

### 5.1 순서

1. **식별**: 대상 디자인 메타데이터 파악
   - 사용 브랜드 (Team Sparta / Sparta Club / Sparta Builders / 기업교육)
   - 콘텐츠 타입 (Content-Driven / Action-Driven / Message-Driven)
   - 레이아웃 포맷 (Mini 3:2 / Flex / Story 1:2 / Feed 4:5)
   - 타겟 매체 (Web / Mobile / SNS)
2. **CRITICAL 검수**: 로고(7항목) + 컬러(5항목) — 12항목 우선 검사
3. **HIGH 검수**: 타이포그래피(6항목) + 그래픽(9항목) — 15항목 검사
4. **MEDIUM 검수**: 레이아웃(5항목) — 5항목 검사
5. **판정**: 심각도별 건수 집계 → PASS/조건부 PASS/FAIL 결정
6. **리포트 생성**: 위 형식에 따라 상세 결과 작성

### 5.2 FAIL 항목 수정 후 재검수

- FAIL 판정 시 수정 방안에 따라 수정 후 재검수 요청 가능
- 재검수 시 이전 리포트의 FAIL 항목만 재확인 (PASS 항목은 생략 가능)
- 재검수 리포트에 이전 판정과 현재 판정을 병기

### 5.3 예외 처리

- 체크리스트에 없는 새로운 유형의 위반 발견 시 → MEDIUM으로 기록 + 피드백 루프에 보고
- 가이드라인 모호 영역 → 판정 보류, `[AMBIGUOUS]` 태그 + 피드백 루프에 보고
- 의도적 가이드라인 이탈 (크리에이티브 디렉터 승인) → `[EXEMPTED]` 태그 + 사유 기록

## 7. 사내 평가 체크리스트

> 기술 검수(32항목)와 별도로 운영되는 **종합 퀄리티 평가**.
> 기술 검수가 "규칙을 지켰는가"를 판단한다면, 사내 평가는 "좋은 디자인인가"를 판단한다.
> 검수 에이전트는 기술 검수 후 사내 평가를 추가로 수행하여 **콘텐츠 등급**을 부여한다.

### 7.1 평가 등급 정의

| 등급 | 조건 | 설명 |
|------|------|------|
| 🔴 **미달** | `필수` 항목 중 1개 이상 FAIL | 게시 불가. 필수 항목 수정 필요 |
| 🟡 **기본** | `필수` 전체 PASS, `중요` 일부만 PASS | 게시 가능하나 개선 권장 |
| 🟢 **우수** | `필수` + `중요` 전체 PASS | 브랜드 품질 기준 충족 |
| 😎 **미침(p)** | 🟢 달성 + `😎` 3개 중 2개 이상 PASS | 탁월한 성과. 팀 레퍼런스 등록 대상 |

### 7.2 퀄리티 영역

#### `QUAL-01` 가독성 및 접근성 — `필수`

모바일/다양한 디바이스에서 콘텐츠가 명확히 보이는가?

| 세부 기준 | 정량 수치 | 측정 방법 |
|-----------|----------|-----------|
| 본문 최소 폰트 크기 | Body ≥ 14px, Caption ≥ 11px | CSS font-size 값 확인 |
| 제목 최소 폰트 크기 | Head ≥ 24px, Title ≥ 18px | CSS font-size 값 확인 |
| 텍스트-배경 명도 대비 (본문) | ≥ 4.5:1 (WCAG AA) | 텍스트 색상과 배경 색상의 대비율 계산 |
| 텍스트-배경 명도 대비 (대형 18px+) | ≥ 3:1 (WCAG AA) | 텍스트 색상과 배경 색상의 대비율 계산 |
| CTA 터치 영역 | ≥ 44 × 44px (모바일 타겟 시) | CTA 버튼의 width × height |
| 핵심 콘텐츠 Safe Area 내 배치 | 로고, 타이틀, CTA가 Safe Area 100% 내 | 포맷별 Safe Area 기준 확인 |

**PASS 조건**: 위 6개 세부 기준 전부 충족

#### `QUAL-02` 레이아웃 완성도 — `필수`

그리드, 간격, 타이포그래피 마감이 디테일하게 정리되었는가?

| 세부 기준 | 정량 수치 | 측정 방법 |
|-----------|----------|-----------|
| 4px 그리드 정렬 | 모든 요소의 margin/padding이 4px 배수 | CSS 스페이싱 토큰(`--space-*`) 사용 여부 |
| 동일 레벨 요소 간격 일관성 | 간격 편차 ≤ 4px | 같은 역할의 요소 간 간격 비교 |
| 타이포 위계 크기 순서 | Head > Title > SubTitle > Body > Caption | 각 레벨의 font-size 비교 |
| 좌우 여백 균형 | 좌우 마진 차이 ≤ 8px (의도적 비대칭 제외) | padding-left vs padding-right |
| 요소 정렬 | 텍스트 요소가 동일 기준선(좌측 또는 중앙)에 정렬 | 텍스트 블록의 text-align/position 확인 |

**PASS 조건**: 위 5개 세부 기준 전부 충족

#### `QUAL-03` 콘텐츠 몰입도 — `중요`

디자인이 콘텐츠의 목적을 방해하지 않고 집중력을 높여주는가?

| 세부 기준 | 정량 수치 | 측정 방법 |
|-----------|----------|-----------|
| 시각적 진입점 명확 | 가장 큰 텍스트/그래픽이 화면 상위 60% 내 배치 | 요소의 y 좌표 확인 |
| 핵심 메시지 시각 가중치 | Title이 모든 텍스트 중 최대 font-size × font-weight | 각 텍스트의 크기·굵기 곱 비교 |
| 텍스트-그래픽 분리 | 텍스트 위에 그래픽 직접 겹침 없음 (Frame Type 사용 시 제외) | 레이어 구조 확인 |
| 정보 밀도 적절 | TYPE-A: ≤ 6개 텍스트 요소 / TYPE-B: ≤ 4개 / TYPE-C: ≤ 3개 | 화면 내 텍스트 블록 수 카운트 |
| CTA 즉시 식별 (TYPE-A/B만) | CTA 버튼의 배경색이 주변과 ≥ 3:1 대비 | CTA 배경색과 주변 배경색 대비율 |

**PASS 조건**: 위 5개 세부 기준 중 4개 이상 충족

### 7.3 브랜드 영역

#### `BRAND-01` 가이드라인 준수 — `필수`

로고, 컬러, 폰트 사용 규정을 지켰는가?

| 세부 기준 | 정량 수치 | 측정 방법 |
|-----------|----------|-----------|
| 기술 검수 판정 | PASS 또는 조건부 PASS | 섹션 2 판정 기준: CRITICAL 0건 AND HIGH ≤ 2건 |

**PASS 조건**: 기술 검수(32항목)에서 FAIL이 아닐 것. 즉, PASS 또는 조건부 PASS

#### `BRAND-02` 일관된 경험 제공 — `중요`

다른 채널/매체의 콘텐츠와 함께 보았을 때 이질감이 없는가?

| 세부 기준 | 정량 수치 | 측정 방법 |
|-----------|----------|-----------|
| 브랜드별 톤앤매너 부합 | 브랜드 정의 톤과 일치 | Team Sparta=공식·신뢰, Club=활기·도전, Builders=전문·협력, 기업교육=신뢰·효율 |
| 컬러 팔레트 일관 | 동일 캠페인 내 동일 Pairing 계열 사용 | 사용 컬러가 동일 Primary/Secondary/Extended 세트에 속하는지 |
| 타이포 위계 패턴 일관 | Head/Title/Body의 서체·Weight·크기 패턴이 동일 채널 기존 콘텐츠와 일치 | 기존 콘텐츠 대비 비교 (비교 대상 없으면 자동 PASS) |
| 그래픽 스타일 통일 | Frame/Pattern/Tag 유형 혼용 없음 (단일 디자인 내) | 한 디자인에서 사용된 키 비주얼 유형 수 = 1 |

**PASS 조건**: 위 4개 세부 기준 중 3개 이상 충족. 비교 대상이 없는 첫 콘텐츠는 자동 PASS

#### `BRAND-03` 혁신성 — `😎`

브랜드를 확장시키는 새로운 표현형식/기법을 시도했는가?

| 세부 기준 | 판정 | 설명 |
|-----------|------|------|
| 새로운 레이아웃 구성 | Y/N | 기존 포맷의 요소 배치를 가이드라인 범위 내에서 새롭게 구성 |
| 창의적 컬러 조합 | Y/N | Extended Pairing 중 기존에 잘 사용하지 않던 조합 활용 |
| 그래픽 모티프 새 적용 | Y/N | Potential Cube 프레임/패턴을 기존과 다른 방식으로 활용 |
| 새로운 텍스트 표현 | Y/N | 타이포 위계를 가이드라인 내에서 실험적으로 구성 |

**PASS 조건**: 위 4개 중 1개 이상 Y이며, 가이드라인 위반 없이 달성

### 7.4 그로스 영역

#### `GROWTH-01` 지표 개선 가능성 — `😎`

작업 이후 관련 지표가 상승할 가능성이 있는가?

| 세부 기준 | 정량 수치 | 측정 방법 |
|-----------|----------|-----------|
| CTA 명확성 (TYPE-A/B) | CTA 버튼이 존재하고 시각적으로 가장 높은 대비 | CTA 배경-텍스트 대비율이 디자인 내 최고 |
| 행동 유도 문구 | CTA 텍스트가 구체적 행동 동사 포함 | "자세히 보기", "지금 신청" 등 동사형 CTA |
| 추적 가능 구조 | 링크에 UTM 또는 트래킹 파라미터 삽입 가능 | `<a>` 태그 href에 파라미터 추가 가능한 구조 |
| A/B 테스트 용이 | 핵심 변수(카피, CTA, 컬러)가 CSS 변수 또는 별도 영역으로 분리 | CSS 변수 사용 + class 기반 구조 |

**PASS 조건**: 위 4개 중 3개 이상 충족

#### `GROWTH-02` 생산성 개선 — `😎`

재사용 가능한 템플릿/모듈로 제작되었는가?

| 세부 기준 | 정량 수치 | 측정 방법 |
|-----------|----------|-----------|
| CSS 변수 활용률 | HEX 직접 사용 0건 (100% 토큰 참조) | CSS 코드에서 `#` 색상값 직접 사용 횟수 |
| 컴포넌트 분리 | 역할별 class 분리 (`.banner-title`, `.cta-button` 등) | class 이름이 역할 기반인지 확인 |
| 콘텐츠 교체 용이 | 텍스트/이미지만 변경하면 새 배너 생성 가능 | 구조 코드와 콘텐츠 코드의 분리도 |
| 레이어 주석 | `<!-- Layer N: ... -->` 주석으로 구조 문서화 | 주석 존재 여부 |

**PASS 조건**: 위 4개 중 3개 이상 충족
